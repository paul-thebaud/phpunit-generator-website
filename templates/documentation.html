Web application - Documentation

Important

DO NOT USE THIS PACKAGE WITHOUT CHECKING GENERATED TESTS!
You must check and complete all tests you generate, including the most basic methods.
Files to parse must declare ONE class, abstract class, trait or interface to be accepted.

Interface

A few actions are available on the main page. You can read this documentation and customize your settings on each page.
There is also the CLEAR button, which will refresh the page.

Configuring your generation

With the button CUSTOMIZE in the top right corner of the application, you can change a few options:

  • Automatic getters / setters tests: If it is activated, it will auto-detect getters and setters, and generate tests for.
  • Author: If it is set, add a "author" tag on the tests class PHP documentation.

NB: On each configuration changes, the application will save it on your browser local storage.

Generating your tests

There is two ways to generate your unit tests:

  • Upload your file: You upload your file (no file will be saved) and it generate tests.
  • Write your code: You paste (and maybe edit) your code and you launch the tests generation.

Once your tests are generated, you can download as a file, copy the code to clipboard or rerun the generation (useful to try the generation configuration!).

Advanced

You can also have annotation in your PHP files: for more details, you can refer to this section of the PHPUnit Generator package documentation.
